About

What is DUON?

DUON (Dynamic Ultrasonic Operations & Navigations) is a 4-wheel drive autonomous mobile robot designed and built as a university engineering project. The robot can be wirelessly controlled from any device — phone, tablet, or PC — through a browser-based dashboard with no app installation required.

Using three ultrasonic sensors, DUON detects obstacles in real time and can navigate a room autonomously. An A* pathfinding algorithm computes the optimal route between two points on a live map, which the robot then follows — stopping and replanning if unexpected obstacles appear.

The entire system — from embedded firmware on two ESP32 microcontrollers to a Python web server to a responsive web dashboard — was designed, wired, coded, and tested from scratch by a two-person team.

The Problem We Solved

Industrial Autonomous Mobile Robot (AMR) systems are highly effective but prohibitively expensive and complex for educational or small-scale use. Academic prototypes, on the other hand, are often too simplified to demonstrate real-world autonomous behaviour.


There was a clear gap: a need for a functional, low-cost, modular autonomous robot that a small team could build with standard components — while still achieving genuine autonomous navigation, real-time sensor processing, and a production-quality control interface. DUON was built to fill that gap.

Architecture

How It Works

[ Browser / Phone / Tablet ] ↕ WebSocket (real-time JSON) [ Python Server on Laptop ] ↕ TCP Socket ↕ TCP Socket [ ESP32 #1 ] [ ESP32 #2 ] [ Left Motors ] [ Right Motors ] [ All 3 Sonars ]

The Layers

The Hardware Layer: Two ESP32 microcontrollers handle the low-level hardware. ESP32 #1 drives the left-side motors and reads all three ultrasonic sensors, streaming distance data wirelessly every 100ms. ESP32 #2 drives the right-side motors. Each controller runs a lightweight TCP server on the robot, waiting for commands from the Python server.

The Server Layer: A Python FastAPI server runs on a laptop on the same Wi-Fi network. It bridges the browser and the robot — forwarding drive commands as single characters to both ESPs via TCP sockets, and broadcasting sensor data and system events to all connected browsers via WebSocket in real time.

The Dashboard Layer: A single-page web application runs in any browser. It provides manual drive controls, live sonar gauges, autonomous mapping, A* navigation, map management, connection monitoring, and a real-time terminal log — all updating live without page reloads.

Hardware Components

  • 2× ESP32 (USB-C) Microcontrollers / Wi-Fi
  • 2× BTS7960 High-current H-Bridge (43A)
  • 4× DC Gearmotor 4WD drive motors with encoders
  • 3× HC-SR04 Left (45°), Front (0°), Right (45°) sonars

Outcomes

Results & Learnings

✅ Autonomous Navigation

A* pathfinding implemented successfully. The robot consistently identified feasible paths between start and goal locations. Obstacle inflation (~30cm safety margin) prevented collisions, and dynamic replanning handled unexpected obstacles.

✅ Real-Time Mapping

The robot successfully generated coherent, real-time representations of indoor spaces as it moved. Wall boundaries and obstacle clusters were identifiable in the map output using three low-cost HC-SR04 sensors.

Challenges Faced

  • Running two BTS7960 drivers from one ESP32 caused reliability issues (solved by dedicating one ESP per driver).
  • HC-SR04 sonar readings were noisy due to 5V output on 3.3V GPIO (mitigated with multi-stage software filtering).
  • Position tracking without encoders drifts over time (currently handled with dead-reckoning).
  • WebSocket broadcasting from background threads in async Python required thread-safe dispatch.

What We Learned

  • Embedded systems, TCP/IP networking, and web technologies can be integrated into a single coherent product when designed layer by layer.
  • Software signal filtering can heavily compensate for hardware limitations in low-cost sensors.
  • A browser-based interface dramatically lowers the barrier to access — no app install, works on any device.
  • A* pathfinding is computationally lightweight enough for real-time planning on a Python laptop.
