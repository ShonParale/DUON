DUON

Dynamic Ultrasonic Operations Navigation

An autonomous 4-wheel-drive industrial transportation robot capable of manual remote control and fully autonomous room mapping using ultrasonic sensors.

Purpose

DUON is designed for autonomous indoor navigation and obstacle mapping in industrial or warehouse environments. It demonstrates real-time sensor fusion, dual-controller architecture, and wireless remote operation over a local network.

  • ◯ Autonomous room mapping
  • ◯ Wireless remote control
  • ◯ Real-time obstacle detection
  • ◯ Edge-hosted web dashboard
🎦

Demonstration

DUON was demonstrated navigating a real-world corridor, successfully avoiding obstacles detected by three ultrasonic sensors while its live 2D map updated in real-time on the dashboard, accessible from a phone on the same Wi-Fi network.

The bot was controlled remotely with zero physical access to the hardware — the Raspberry Pi served the dashboard automatically on boot, and the ESPs connected on power-up.

🌐 How It Works

01

Dual ESP32 Architecture

ESP32 #1 controls the left-side motors and all three HC-SR04 ultrasonic sensors. ESP32 #2 controls the right-side motors. Both listen for TCP commands on port 8080.

02

Python Bridge Server

The Raspberry Pi (or your laptop) runs this dashboard server. It maintains persistent TCP sockets to both ESPs, forwarding drive commands and streaming sensor data back.

03

WebSocket Dashboard

This web dashboard connects via WebSocket to the Python server. Commands you send (WASD, joystick, etc.) are relayed to both ESPs simultaneously in real-time.

04

Autonomous Mapping

The sonar mapper uses a wall-following algorithm with configurable pulse/smooth modes, building a live 2D top-down map plotted on the canvas as the robot explores.

📚 User Guide

🔌 Network Page

  • IP Address: Enter the IP shown in Arduino Serial Monitor when ESP32 boots on Wi-Fi.
  • Connect / Disconnect: Click to open or close a TCP socket to each ESP on port 8080.
  • QR Code: Scan with phone to instantly open dashboard. Updates automatically for Cloudflare tunnels.
  • Connection Quality: Shows WebSocket latency (browser↔server) and time since last ESP data packet.

🕹 Drive Page

  • D-Pad: Click/tap or use WASD / Arrow keys. Space = Stop.
  • Numpad: Click buttons or use Numpad 8462. 5 = Stop.
  • Omni Joystick: Drag the glowing knob in any direction. Release to stop. Works with touch.
  • Dual Axis: Left stick = forward/backward. Right stick = left/right. Release any stick to stop.
  • Auto E-Stop: Automatically stops robot when any sonar reads ≤ 10 cm.

🗺 Mapper Page

  • Start Mapping: Robot begins autonomous navigation. It avoids walls and maps free space.
  • Pulse mode: Robot stops, reads sonar, then moves. More accurate mapping.
  • Smooth mode: Robot moves continuously, reacting to obstacles in real time.
  • Map colors: Red = Wall   Cyan = Path   Green = Free space   Yellow diamond = Bot.
  • Canvas controls: Scroll to zoom, click-drag to pan, double-click to reset view.

● Status Bar

  • ESP1/ESP2 dots: Green pulsing = connected. Red = offline.
  • STOP button: Instantly sends stop command to both ESPs (manual emergency stop).
  • AUTO button: Arm/disarm the automatic proximity-based emergency stop.
  • F: xx cm: Live front sonar distance (updates in real-time when ESP1 connected).
  • MAP: ON/OFF: Shows whether auto-mapping is currently running.
  • LIVE indicator: Green = WebSocket connected. Flashes with latency in ms.
